#### 2.1 GET /api/v1/admin/users
**File:** `app/api/v1/admin/users/route.ts` (110 lines)

**Features:**
- List all users with pagination (default 50, max 100 per page)
- Search by email or full name (ilike pattern matching)
- Filter by role (admin/user/guest)
- Filter by status (active/inactive/suspended/pending_verification)

- Stats: count by role, count by status

**Query Parameters:**
- `page` (default: 1)
- `limit` (default: 50, max: 100)

- `sortOrder` (asc/desc, default: desc)

```json

    "limit": 50,

  "stats": {

    "byStatus": { "active": 140, "suspended": 10 }
```

### Database Integration

**Tables Used:**
- `users` (role, status, profile fields)
- `user_sessions` (active sessions tracking)
- `audit_logs` (security audit trail)

**Schema (from migration 20260207_create_users_and_auth.sql):**

```sql
-- Users table
CREATE TABLE users (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email TEXT NOT NULL UNIQUE,
  full_name TEXT,
  avatar_url TEXT,
  role user_role NOT NULL DEFAULT 'user',
  status user_status NOT NULL DEFAULT 'pending_verification',
  metadata JSONB DEFAULT '{}'::jsonb,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  last_login_at TIMESTAMPTZ
);

-- User sessions table
CREATE TABLE user_sessions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  session_token TEXT NOT NULL UNIQUE,
  user_agent TEXT,
  ip_address INET,
  last_activity_at TIMESTAMPTZ DEFAULT now(),
  expires_at TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Audit logs table
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE SET NULL,
  action TEXT NOT NULL,
  resource_type TEXT,
  resource_id TEXT,
  ip_address INET,
  user_agent TEXT,
  metadata JSONB DEFAULT '{}'::jsonb,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

**RLS Policies:**
- Users can view own profile
- Admins can view all profiles
- Only service_role can modify users (backend routes)
- Similar policies for sessions and audit_logs
